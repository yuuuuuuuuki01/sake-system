# SHDEN.DAT フィールドマップ（確定済み）

> **重要**: このファイルはバイナリ解析で確定したオフセットです。
> 変更前に必ずここを参照してください。

## ファイル構造

- Magic ISAM B-Tree フォーマット
- ファイルヘッダ: 0x200 (512) バイト
- レコードサイズ: `struct.unpack_from("<H", data, 0x18)[0]` で取得（現在 497 バイト）
- 全スロット数: `(ファイルサイズ - 0x200) // record_size`（現在 約 93 万スロット）

## データノードの判定方法

**2 条件を両方満たすレコードのみがデータノード（実際の売上伝票ヘッダ）**

1. `rec[234:242]` が `20[12]\d(0[1-9]|1[0-2])(0[1-9]|[12]\d|3[01])` にフルマッチ
2. `rec[330:336]` が数字のみの得意先コード（例: `"012345"`）

→ 条件 1 だけでは B-Tree インデックスノードも通過してしまう。**条件 2 は必須。**

データノードは全スロットの **約 0.01%** のみ（93 万スロット中 ~117 件）。これは正常。

## 確定フィールドマップ

| オフセット | サイズ | 型 | 内容 |
|-----------|-------|----|------|
| @50–52 | 3B | ASCII | 担当コード |
| @234–241 | 8B | ASCII `YYYYMMDD` | 計上日（データノード判定に使用） |
| **@260–267** | 8B | ASCII `YYYYMMDD` | **請求日**（`@258` は誤り） |
| @276–277 | 2B | ASCII | 締日 |
| **@330–335** | 6B | ASCII | **得意先コード**（データノード判定に使用） |
| @337–376 | 40B | CP932 | 得意先名 |
| @390–429 | 40B | CP932 | 備考 |
| **@444–455** | 12B | ASCII | **伝票番号**（空の場合あり → フォールバック） |

### 伝票番号の取得方法

```python
doc_raw = rec[444:456].decode("ascii", errors="replace").strip()
if doc_raw and doc_raw.isdigit():
    doc_no = doc_raw
else:
    doc_no = f"S{slot}"   # スロット番号をフォールバック（decoder_sales.py 方式）
```

**`@326` の int32 を伝票番号に使ってはいけない。** B-Tree インデックスポインタであり、
得意先コードありのスロットでは有効な整数に見えるが伝票番号ではない。

## よくある間違い

| 間違い | 正しい |
|--------|--------|
| `@330` フィルタを外す → B-Tree インデックスノードを大量誤検出 | **必ず** `@330` 得意先コードでフィルタ |
| 請求日を `rec[258:266]` から取る | **`rec[260:268]`** を使う |
| `@326` の int32 を伝票番号として使う | `@444` ASCII → スロット番号フォールバック |
| 日付のみで 498 件見つかる → 「全部データノード」と誤認 | 得意先コードフィルタ後は ~117 件が正常 |

## SHTOR.DAT との違い

SHTOR.DAT（売上明細）は SHDEN.DAT とは別のファイルで、レコード構造も異なる。
SHTOR.DAT の日付は固定オフセットではなく `DATE_RE.finditer(rec)` で検索する。
